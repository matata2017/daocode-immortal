# 心魔复习系统 (Demon Review System)

> **Status**: Designed
> **Author**: User + Claude Code
> **Last Updated**: 2026-03-30
> **Implements Pillar**: 成长可见, 实用至上

## Overview

**心魔复习系统**负责管理玩家复习错题的完整流程，包括心魔选择、复习答题、结果反馈、状态更新。

系统从错题记录系统获取心魔列表，按优先级排序展示。玩家选择心魔后进入复习答题，答对减少错误计数，答错增加错误计数。连续答对指定次数后心魔被斩除，触发庆祝反馈。

**为什么需要这个系统**: 没有复习流程，心魔只是"记录"而非"可行动"的对象。系统化的复习流程让玩家有明确路径攻克心魔，形成"答错→复习→斩除"的完整闭环，真正实现"错题变强"的成长体验。

## Player Fantasy

玩家期望体验：

**1. 针对性复习的效率感**
- "我知道自己哪里薄弱，直奔主题"
- 心魔按优先级排序，最需要复习的排在前面
- 不必盲目刷题，每一题都有明确目的

**2. 攻克心魔的仪式感**
- 进入心魔场景有独特的氛围（暗色调、压迫感BGM）
- 斩除心魔有明确的视觉反馈（心魔消散动画、清脆音效）
- 心魔数量减少带来直观的成就感

**3. 避免重复犯错的安心感**
- 曾经错过的知识点被系统化追踪
- 复习后真正掌握（连续答对）才能斩除
- 不必担心"这次记住了下次又忘"

## Detailed Design

### Core Rules

#### 心魔复习入口条件
| 条件 | 要求 |
|------|------|
| 心魔列表非空 | `wrong_questions.size() > 0` |
| 场景已解锁 | 玩家已进入心魔场景（心魔洞/心魔殿） |

#### 复习流程
```
1. 展示心魔列表 → 按优先级排序（priority_score 降序）
2. 玩家选择心魔 → 加载题目详情
3. 进入复习答题界面 → 显示题面
4. 玩家作答 → 判定对错
5. 答对 → wrong_count -= 1, 检查是否斩除
6. 答错 → wrong_count += 1, 复习失败，返回列表
7. 斩除检查 → 若连续答对次数达标，移除心魔，触发斩除动画
8. 返回心魔列表或继续复习下一题
```

#### 心魔斩除条件（继承自错题记录系统）
```
can_exorcise = (consecutive_correct_count >= required_correct_count)

where:
  consecutive_correct_count = 当前心魔连续答对次数
  required_correct_count = 2（默认，来自错题记录系统）
```

**注意**: 连续答对指的是**同一心魔**的连续答对。如果中途复习其他心魔或退出，当前心魔的连续答对计数**不重置**，但**不继承**到下次进入。

#### 复习答题与普通修炼的区别
| 维度 | 心魔复习 | 普通修炼 |
|------|----------|----------|
| 题目来源 | 仅限心魔列表中的错题 | 题库随机/按规则抽取 |
| 答对奖励 | 减少错误计数，可能斩除心魔 | 获得修为值 |
| 答错惩罚 | 增加错误计数，心魔加深 | 修为为0，无惩罚 |
| 修为获取 | **无**（复习不计修为） | 按难度计算 |

**设计意图**: 复习是"补短板"，不应获得修为奖励。玩家的收益是"清除心魔"本身。

## Formulas

### 心魔排序（继承自错题记录系统）
```
priority_score = wrong_count × wrong_weight + days_since_last_review × days_weight

where:
  wrong_count = 累计答错次数
  days_since_last_review = 距上次复习的天数（从未复习则用距首次答错的天数）
  wrong_weight = 10（默认）
  days_weight = 2（默认）
```

**排序规则**: priority_score 降序，分数高的排在前面（更急需复习）。

### 复习结果处理
```
CASE review_result:
  correct:
    demon.wrong_count -= 1
    demon.last_review_at = now()
    IF demon.wrong_count == 0:
      exorcise_demon(demon)
  wrong:
    demon.wrong_count += 1
    demon.last_wrong_at = now()
    // 不更新 last_review_at（复习失败不算复习成功）
```

### 连续答对计数追踪
```
consecutive_correct = {
  demon_id: int  // 每个心魔单独追踪
}

// 每次复习答对时
consecutive_correct[demon_id] += 1

// 每次复习答错时
consecutive_correct[demon_id] = 0

// 斩除检查
IF consecutive_correct[demon_id] >= required_correct_count:
  exorcise_demon(demon_id)
  consecutive_correct[demon_id] = 0  // 清除计数
```

### 复习时长估算
```
estimated_review_time = demon_count × avg_time_per_question

where:
  demon_count = 心魔总数
  avg_time_per_question = 60秒（中等难度题的平均答题时间）

# 示例：10个心魔
# 10 × 60 = 600秒 ≈ 10分钟
```

**用途**: 在心魔列表顶部显示"预计复习时间"，帮助玩家规划。

## Edge Cases

| 边缘情况 | 处理方式 | 影响范围 |
|----------|----------|----------|
| **心魔列表为空** | 显示"暂无心魔"空状态提示，压迫感显示0% | 心魔UI系统 |
| **复习中途退出** | 当前心魔的连续答对计数保留（不重置），进度不丢失 | 复习流程 |
| **复习同一心魔答错** | `wrong_count += 1`，连续答对计数重置为0，心魔加深 | 复习流程 |
| **心魔在复习中被斩除** | 移除心魔，触发斩除动画，返回列表 | 复习流程 |
| **复习时题目被删除** | 跳过该题，显示"题目已失效"，从心魔列表移除 | 题库维护 |
| **连续答对计数已达但未斩除** | 下次进入自动检查并触发斩除 | 复习流程 |
| **心魔数量超过50个** | 列表正常显示，但提示"心魔过多，建议优先复习高危心魔" | 心魔UI系统 |
| **复习时存档损坏** | 当前复习进度丢失，显示错误提示，返回主菜单 | 存档系统 |
| **复习答题超时** | 按答错处理，增加 `wrong_count` | 复习流程 |
| **心魔复习场景未解锁** | 隐藏入口，显示解锁条件（如"达到练气期"） | 场景管理系统 |

## Tuning Knobs

| Knob | 类型 | 默认值 | 范围 | 影响范围 |
|------|------|--------|------|----------|
| `required_correct_count` | int | 2 | 1-5 | 斩除心魔需要的连续答对次数 |
| `recent_weight` | int | 2 | 1-5 | 优先级公式中"未复习天数"的权重 |
| `wrong_weight` | int | 10 | 5-20 | 优先级公式中"错误次数"的权重 |
| `max_review_time_per_question` | int | 120 | 60-300 | 单题最长答题时间（秒） |
| `show_estimated_time` | bool | true | true/false | 是否在列表顶部显示预计复习时间 |

### 调参指南

**`required_correct_count`（斩除所需连续答对次数）**
- **调低**: 斩除更容易，玩家成就感更强，但可能不够扎实
- **调高**: 斩除更难，确保真正掌握，但可能挫败感强
- **观察指标**: 斩除成功率、重复犯错率

**`recent_weight` vs `wrong_weight`（优先级权重）**
- **提高 recent_weight**: 长时间未复习的题优先级更高，防止遗忘
- **提高 wrong_weight**: 多次犯错的题优先级更高，聚焦顽固问题
- **观察指标**: 玩家复习顺序选择（是否按推荐顺序）

**`max_review_time_per_question`（单题超时）**
- **调低**: 强制造迫思考，增加紧张感
- **调高**: 给予更多思考时间，降低焦虑
- **观察指标**: 答题超时率、平均答题时间

## 附录：复习流程示例

```
玩家进入心魔洞场景
↓
系统展示心魔列表（按优先级排序）:
  1. [高危] q_concurrency_015 - 答错3次 - 5天未复习
  2. [高危] q_spring_042 - 答错3次 - 2天未复习
  3. [中危] q_jvm_008 - 答错2次 - 1天未复习
  4. [低危] q_collections_023 - 答错1次 - 刚答错
↓
玩家选择 q_concurrency_015
↓
系统加载题目详情，显示题面
↓
玩家作答
↓
【情况A：答对】
  - wrong_count: 3 → 2
  - 连续答对计数: 1
  - 检查斩除: 1 < 2，不斩除
  - 继续复习同一题
  - 再次答对
  - wrong_count: 2 → 1
  - 连续答对计数: 2
  - 检查斩除: 2 >= 2，斩除！
  - 触发斩除动画
  - 从心魔列表移除
  - 返回列表

【情况B：答错】
  - wrong_count: 3 → 4
  - 连续答对计数: 0
  - 复习失败，返回列表
  - 心魔加深，优先级提高
```
