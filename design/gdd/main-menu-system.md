# 主菜单系统 (Main Menu System)

> **Status**: Designed
> **Author**: User + Claude Code
> **Last Updated**: 2026-03-30
> **Implements Pillar**: 碎片友好

## Overview

**主菜单系统**负责游戏启动后的主界面，包括开始游戏、继续游戏、设置、关于等入口，以及玩家基础信息展示。

**为什么需要这个系统**: 主菜单是玩家进入游戏的第一印象，需要清晰、简洁、快速进入游戏。

## Player Fantasy

玩家期望体验：

**1. 快速进入游戏**
- 打开游戏即可继续上次的进度
- 一键开始修炼，无需复杂流程
- 碎片时间友好

**2. 清晰的信息展示**
- 当前境界、修为一目了然
- 学习进度可视化
- 心魔数量提醒

**3. 便捷的设置入口**
- 音量设置快速访问
- 不打断游戏流程

#### 菜单项功能

| 菜单项 | 功能 | 跳转 |
|--------|------|------|
| **继续修炼** | 进入修炼场景 | 修炼场景系统 |
| **修炼场景** | 进入修炼场景 | 修炼场景系统 |
| **心魔洞** | 进入心魔复习 | 心魔UI系统 |
| **BOSS面试** | 选择BOSS进行面试 | BOSS选择界面 |
| **学习分析** | 查看学习统计 | 学习分析系统 |
| **设置** | 打开设置界面 | 设置系统 |
| **关于** | 显示游戏信息 | 关于界面 |

### States and Transitions

```
┌─────────────────┐
│   游戏启动        │
└────────┬────────┘
         │ 加载存档
         ↓
┌─────────────────┐
│   主菜单展示      │  显示玩家信息
└────────┬────────┘
         │ 点击菜单项
         ↓
┌─────────────────┐
│   场景切换        │  进入目标场景
└─────────────────┘
```

### Interactions with Other Systems

| 系统 | 数据流向 | 接口 |
|------|----------|------|
| **场景管理系统** | 写入 | `change_scene()` |
| **门派数据系统** | 读取 | `get_faction_info()` |
| **存档系统** | 读取 | `get_save_data()` |
| **设置系统** | 写入 | `open_settings()` |

## Edge Cases

| 边缘情况 | 处理方式 |
|----------|----------|
| **无存档** | 显示"开始游戏"按钮 |
| **存档损坏** | 显示错误，允许重新开始 |
| **首次启动** | 显示新手引导入口 |
