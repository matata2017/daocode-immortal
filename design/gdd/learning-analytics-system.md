# 学习分析系统 (Learning Analytics System)

> **Status**: Designed
> **Author**: User + Claude Code
> **Last Updated**: 2026-03-30
> **Implements Pillar**: 成长可见, 实用至上

## Overview

**学习分析系统**负责收集、计算、可视化玩家的答题数据，提供多维度统计分析，帮助玩家了解自己的学习进度和薄弱环节。

系统从存档系统读取答题记录，从题库/门派系统获取题目元数据（知识点、难度、领域），计算各类统计指标，并通过可视化图表展示。玩家可以查看知识点掌握度雷达图、错误热力图、正确率趋势等。

**为什么需要这个系统**: "成长可见"是游戏核心支柱。没有详细分析，玩家只知道"我做了题"但不知道"我哪里强、哪里弱"。系统化分析让玩家清晰看到进步轨迹，获得持续学习的动力。对于面试准备场景，明确的薄弱项分析让复习更有针对性。

## Player Fantasy

玩家期望体验：

**1. 数据驱动的掌控感**
- "我知道自己哪里强、哪里弱，不盲目刷题"
- 一眼看清知识点掌握度雷达图，薄弱领域一目了然
- 错误热力图让"薄弱项"变得具体可行动

**2. 进步可见的成就感**
- 正确率趋势线向上走，每一道题都有意义
- 从"陌生"到"熟悉"的知识点状态变化清晰可见
- 心魔斩除数量累计，攻克障碍的历史记录

**3. 复习规划的科学感**
- 基于数据推荐复习重点，而非凭感觉
- "本周建议复习：并发编程"有数据支撑
- 学习报告让面试准备更系统化

## Detailed Design

### Core Rules

#### 统计维度

| 维度 | 数据来源 | 计算方式 | 展示形式 |
|------|----------|----------|----------|
| **总体正确率** | `answered_questions` | `total_correct / total_answered` | 百分比 + 数字 |
| **知识点正确率** | `answered_questions` + 题目tags | 按tag分组计算正确率 | 雷达图 |
| **知识点掌握度** | `answered_questions` + 题目tags | `(首次答对 + 复习答对) / 题目总数` | 进度条 |
| **难度正确率** | `answered_questions` + 题目difficulty | 按难度分组计算正确率 | 柱状图 |
| **门派进度** | `faction_mastery` | 各domain的mastery值 | 雷达图 |
| **心魔分布** | `wrong_questions` + 题目tags | 按tag统计心魔数量 | 热力图 |
| **答题时间分布** | `answered_questions` | 每题的答题时长 | 柱状图 |
| **每日趋势** | `answered_questions` | 按日期分组统计 | 折线图 |
| **心魔转化率** | `wrong_questions` | `已斩除数 / 曾有心魔总数` | 数字 + 百分比 |

#### 数据采集

**存档数据结构（来自存档系统）**:
```gdscript
# 答题记录
answered_questions: Dictionary  # {question_id: {correct: bool, answered_at: String, time_spent: float}}

# 心魔记录
wrong_questions: Dictionary  # {question_id: {wrong_count: int, first_wrong_at: String, last_wrong_at: String, last_review_at: String}}

# 门派掌握度
faction_mastery: Dictionary  # {faction_id: {domain_mastery: Dictionary}}
```

**额外采集字段（运行时计算）**:
- `time_spent`: 答题耗时（秒）
- `attempt_count`: 尝试次数（同一题多次答题）
- `first_try_correct`: 首次答题是否正确

#### 统计周期

| 周期 | 范围 | 用途 |
|------|------|------|
| **实时** | 当前会话 | 即时反馈 |
| **今日** | 当天 0:00 - 现在 | 每日总结 |
| **本周** | 本周一 - 现在 | 周报 |
| **本月** | 本月1日 - 现在 | 月报 |
| **全部** | 存档创建至今 | 总体进度 |

## Formulas

### 总体正确率
```
overall_accuracy = total_correct / total_answered × 100%

where:
  total_correct = answered_questions.values().filter(r => r.correct).size()
  total_answered = answered_questions.size()
```

**示例**: 200题答对160题 → `160/200 × 100% = 80%`

### 知识点正确率（按Tag分组）
```
tag_accuracy[tag] = tag_correct[tag] / tag_total[tag] × 100%

where:
  tag_correct[tag] = 答对的题目中包含此tag的数量
  tag_total[tag] = 答过的题目中包含此tag的数量
```

**示例**:
- "concurrency" tag: 答过20题，答对15题 → `15/20 = 75%`
- "spring" tag: 答过30题，答对27题 → `27/30 = 90%`

### 知识点掌握度（含首次答对加分）
```
tag_mastery[tag] = (first_try_correct_count × 1.2 + review_correct_count) / tag_total × 100%

where:
  first_try_correct_count = 首次答对的题目数
  review_correct_count = 复习后答对的题目数（心魔斩除）
  1.2 = 首次答对权重系数（首次答对价值更高）
```

**设计意图**: 首次答对说明真正掌握，复习答对是补短板，首次答对给予1.2倍权重。

### 门派领域掌握度（雷达图）
```
domain_mastery[domain] = Σ(tag_mastery[tag] for tag in domain) / domain_tag_count × 100%

where:
  domain = 门派下的某个领域（如Java门派的"concurrency"领域）
  domain_tag_count = 该领域包含的tag总数
```

### 心魔密度（热力图强度）
```
demon_density[tag] = min(wrong_count_sum[tag] / max_wrong_count, 1.0)

where:
  wrong_count_sum[tag] = 该tag下所有心魔的wrong_count总和
  max_wrong_count = 热力图最大值归一化参数（默认10）
```

**颜色映射**:
- 0.0 - 0.3: 浅红色（低密度）
- 0.3 - 0.6: 中红色（中密度）
- 0.6 - 1.0: 深红色（高密度）

### 每日正确率趋势
```
daily_accuracy[date] = daily_correct[date] / daily_answered[date] × 100%

where:
  daily_correct[date] = 该日答对数量
  daily_answered[date] = 该日答题数量
```

### 心魔转化率
```
demon_conversion_rate = exorcised_count / total_ever_had_demons × 100%

where:
  exorcised_count = 已斩除的心魔数量（累计）
  total_ever_had_demons = 曾经拥有过的心魔总数
```

**示例**: 曾有50个心魔，已斩除35个 → `35/50 = 70%`

### 答题速度评分
```
speed_score = 100 - min(avg_time_per_question / target_time × 100, 100)

where:
  avg_time_per_question = 平均答题时间（秒）
  target_time = 目标答题时间（默认60秒）
```

**示例**: 平均50秒/题 → `100 - 50/60 × 100 = 100 - 83 = 17`（速度评分83分）

## Edge Cases

| 边缘情况 | 处理方式 | 影响范围 |
|----------|----------|----------|
| **无答题记录** | 显示"尚未开始修炼"空状态，正确率显示为 "--" | 图表展示 |
| **某一tag无数据** | 雷达图中该点显示为0%，点击显示"该领域尚未答题" | 雷达图 |
| **某日无答题** | 趋势图中该日跳过，或显示为空点（断点） | 趋势图 |
| **心魔列表为空** | 热力图显示"暂无心魔"，转化率显示100%（无心魔=全部攻克） | 热力图 |
| **题目被删除** | 统计时跳过已删除题目，不计入总数 | 统计计算 |
| **存档损坏/丢失** | 显示"数据加载失败"，所有统计显示 "--" | 整体界面 |
| **答题数量过少（<10题）** | 显示提示"答题数量较少，数据可能不够准确" | 总体概览 |
| **单一门派数据** | 雷达图只显示该门派的domain，其他门派隐藏 | 门派进度 |
| **极短答题时间（<5秒）** | 视为"快速作答"，计入统计但不显示异常 | 答题速度 |
| **极长答题时间（>300秒）** | 视为"暂停作答"，不计入答题速度统计 | 答题速度 |

## Tuning Knobs

| Knob | 类型 | 默认值 | 范围 | 影响范围 |
|------|------|--------|------|----------|
| `first_try_correct_weight` | float | 1.2 | 1.0-2.0 | 知识点掌握度公式中首次答对的权重 |
| `target_answer_time` | int | 60 | 30-120 | 答题速度评分的目标时间（秒） |
| `heatmap_max_density` | int | 10 | 5-20 | 热力图最大密度归一化参数 |
| `trend_chart_days` | int | 30 | 7-90 | 趋势图默认显示的天数 |
| `min_questions_for_accuracy` | int | 10 | 5-20 | 显示正确率的最小答题数 |
| `show_speed_score` | bool | true | true/false | 是否显示答题速度评分 |

### 调参指南

**`first_try_correct_weight`（首次答对权重）**
- **调高**: 首次答对的价值更大，鼓励认真学习而非反复试错
- **调低**: 复习答对的价值更大，鼓励攻克心魔
- **观察指标**: 玩家首次答对率 vs 复习斩除率

**`target_answer_time`（目标答题时间）**
- **调低**: 对速度要求更高，增加紧迫感
- **调高**: 给予更多思考时间，降低焦虑
- **观察指标**: 平均答题时间分布

**`heatmap_max_density`（热力图最大密度）**
- **调低**: 热力图更容易显示"高密度"，红色更常见，增加紧迫感
- **调高**: 热力图显示更温和，减少焦虑
- **观察指标**: 热力图颜色分布（是否过于红/过于白）

**`trend_chart_days`（趋势图天数）**
- **调低**: 显示近期趋势，更聚焦短期进步
- **调高**: 显示长期趋势，更看清整体进步轨迹
- **观察指标**: 趋势图是否过于波动/过于平滑
