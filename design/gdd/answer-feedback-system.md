# 答题反馈系统 (Answer Feedback System)

> **Status**: Designed
> **Author**: User + Claude Code
> **Last Updated**: 2026-03-30
> **Implements Pillar**: 成长可见, 实用至上

## Overview

**答题反馈系统**负责在玩家答题后展示反馈信息，包括正确/错误判定、AI 反馈文本、修为变化、音效播放、动画效果。

系统接收答题结果数据，组合视觉、音频、文字反馈，向玩家呈现完整的答题体验。反馈内容来自回答分析系统，视觉和音频效果由本系统控制。

**为什么需要这个系统**: 没有反馈系统，答题只是"点了选项→看到对错"，缺乏仪式感和学习价值。系统化的反馈让每次答题都有"收获感"，正确时有成就感，错误时有学习机会。

## Player Fantasy

玩家期望体验：

**1. 答对的成就感**
- "我答对了！"的即时满足感
- 视觉：绿色高亮 + 撒花/光芒动画
- 音效：清脆正确的提示音
- 修为增加的数字跳动

**2. 答错的学习感**
- "原来是这样"的恍然大悟
- 视觉：红色提示 + 正确答案展示
- 音效：低沉错误的提示音
- AI 讲解帮助理解

**3. 反馈的节奏感**
- 不拖沓：反馈快速出现
- 不匆忙：有足够时间阅读
- 可跳过：熟悉后可快速跳过

## Detailed Design

### Core Rules

#### 反馈类型

| 类型 | 触发条件 | 视觉 | 音效 | 时长 |
|------|----------|------|------|------|
| **正确反馈** | 答案正确 | 绿色高亮 + 光芒动画 | `sfx_correct.ogg` | 1.5s |
| **错误反馈** | 答案错误 | 红色提示 + 震动 | `sfx_wrong.ogg` | 2.0s |
| **部分正确** | 开放题评分及格 | 黄色 + 部分正确提示 | `sfx_correct.ogg` | 2.0s |
| **优秀反馈** | 开放题评分优秀 | 金色特效 + 优秀徽章 | `sfx_correct.ogg` + 额外音效 | 2.5s |

#### 反馈内容组成

```
反馈界面结构:
┌─────────────────────────────────────────────────────────────┐
│  [判定图标] 答对了！/ 答错了...                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  正确答案: B. HashMap使用数组+链表                           │
│                                                             │
│  AI 讲解:                                                   │
│  HashMap底层确实是数组+链表结构，但在JDK 1.8后...            │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│  修为 +20 ─────────────────────────────────────────────→    │
│                                                             │
│  [继续答题]  [查看详解]                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 反馈流程

```
1. 接收答题结果
   - question_id
   - player_answer
   - verdict (correct/wrong/partial)
   - feedback_text (AI 反馈)
   - explanation (讲解)
   - cultivation_change (修为变化)

2. 判断反馈类型
   - 根据verdict选择反馈模板

3. 播放音效
   - 调用音频系统播放对应音效

4. 显示反馈界面
   - 动画效果
   - 文字内容
   - 修为变化

5. 等待玩家操作
   - 点击"继续"或自动倒计时
   - 或点击"查看详解"

6. 关闭反馈界面
   - 返回答题流程
```

#### 自动跳过规则

| 条件 | 行为 |
|------|------|
| 答对且已看讲解 | 3秒后自动跳过 |
| 答错首次 | 需手动点击"继续" |
| 答错且已复习过此题 | 2秒后自动跳过 |
| 设置中开启"快速模式" | 1秒后自动跳过 |

### States and Transitions

```
┌─────────────────┐
│   空闲状态       │  等待答题完成
└────────┬────────┘
         │ receive_result()
         ↓
┌─────────────────┐
│   音效播放中     │  播放正确/错误音效
└────────┬────────┘
         │ 音效完成
         ↓
┌─────────────────┐
│   动画展示中     │  判定动画（光芒/震动）
└────────┬────────┘
         │ 动画完成
         ↓
┌─────────────────┐
│   内容展示中     │  显示反馈内容
└────────┬────────┘
         │ 玩家操作/自动跳过
         ↓
┌─────────────────┐
│   关闭动画中     │  淡出动画
└────────┬────────┘
         │ 动画完成
         ↓
┌─────────────────┐
│   返回答题流程   │
└─────────────────┘
```

### Interactions with Other Systems

| 系统 | 数据流向 | 接口 |
|------|----------|------|
| **音频系统** | 写入 | `play_sfx("answer_correct/wrong")` |
| **回答分析系统** | 读取 | 接收答题结果数据 |
| **修为计算系统** | 读取 | 接收修为变化数值 |
| **答题修炼系统** | 双向 | 接收结果 → 返回"继续"信号 |
| **BOSS面试系统** | 双向 | 接收结果 → 返回"继续"信号 |
| **设置系统** | 读取 | `animations_enabled`, `skip_exorcise_animation` |

## Formulas

### 反馈时长计算
```
feedback_duration = base_duration + extra_time_for_reading

where:
  base_duration = 1.5s (正确) / 2.0s (错误) / 2.0s (部分正确)
  extra_time_for_reading = len(feedback_text) / reading_speed

  reading_speed = 20 字/秒 (默认阅读速度)
```

**示例**:
- 反馈文本50字 → `50 / 20 = 2.5s` 额外时间
- 总时长 = `1.5 + 2.5 = 4.0s`

### 自动跳过判定
```
should_auto_skip = (
    settings.skip_feedback_enabled OR
    (verdict == "correct" AND has_seen_explanation) OR
    (verdict == "wrong" AND review_count > 0 AND elapsed_time > 2000ms)
)
```

### 修为变化动画
```
cultivation_animation = {
    start_value: previous_cultivation,
    end_value: new_cultivation,
    duration: 500ms,
    easing: "ease_out_quad"
}
```

### 反馈队列处理
```
IF feedback_queue.size() > 1:
    // 多个反馈排队（如连击奖励）
    show_combined_feedback()
ELSE:
    show_single_feedback()
```

## Edge Cases

| 边缘情况 | 处理方式 | 影响范围 |
|----------|----------|----------|
| **反馈文本为空** | 显示默认文本"回答正确/错误" | 反馈内容 |
| **音效播放失败** | 静默继续，不阻塞反馈 | 音频播放 |
| **动画被跳过（设置关闭）** | 直接显示结果，无动画 | 视觉效果 |
| **修变为0（答错）** | 显示"修为不变"，不显示减少 | 修为显示 |
| **快速连续答题** | 反馈队列排队，依次展示 | 反馈流程 |
| **反馈界面被强制关闭** | 保存已读状态，下次不再自动跳过 | 用户体验 |
| **开放题评分失败** | 显示"评分中..."，稍后更新 | 反馈内容 |
| **AI 讲解过长（>500字）** | 显示前300字 + "展开更多"按钮 | 反馈内容 |
| **网络延迟导致反馈慢** | 显示"加载中..."，超时后使用简化反馈 | 网络延迟 |
| **玩家快速点击跳过** | 记录跳过行为，后续缩短自动时长 | 用户体验 |

## Tuning Knobs

| Knob | 类型 | 默认值 | 范围 | 影响范围 |
|------|------|--------|------|----------|
| `correct_feedback_duration` | float | 1.5 | 0.5-3.0 | 正确反馈基础时长（秒） |
| `wrong_feedback_duration` | float | 2.0 | 1.0-4.0 | 错误反馈基础时长（秒） |
| `reading_speed` | int | 20 | 10-40 | 阅读速度（字/秒） |
| `auto_skip_delay_correct` | int | 3000 | 1000-5000 | 正确后自动跳过延迟（毫秒） |
| `auto_skip_delay_wrong` | int | 0 | 0-5000 | 错误后自动跳过延迟（0=不自动） |
| `max_feedback_text_length` | int | 300 | 100-500 | 最大显示文本长度 |
| `animation_enabled_default` | bool | true | true/false | 默认开启动画 |

### 调参指南

**`reading_speed`（阅读速度）**
- **调低**: 反馈停留更长，适合慢速阅读者
- **调高**: 反馈更快，适合快速阅读者
- **观察指标**: 玩家是否频繁手动跳过

**`auto_skip_delay_correct`（正确后自动跳过）**
- **调低**: 更快进入下一题，节奏快
- **调高**: 有更多时间看到修为变化
- **观察指标**: 玩家对节奏的反馈

**`max_feedback_text_length`（最大文本长度）**
- **调低**: 界面更简洁，但可能截断重要内容
- **调高**: 显示完整内容，但可能过长
- **观察指标**: "展开更多"点击率

## 附录：反馈界面设计规范

### 正确反馈
```
颜色方案：
- 背景：#E8F5E9（浅绿色）
- 边框：#4CAF50（绿色）
- 图标：✓ 白色对勾
- 文字：#2E7D32（深绿色）

动画效果：
1. 缩放弹出（0.8 → 1.0）
2. 光芒从中心向外扩散
3. 修为数字跳动增长
```

### 错误反馈
```
颜色方案：
- 背景：#FFEBEE（浅红色）
- 边框：#F44336（红色）
- 图标：✗ 白色叉号
- 文字：#C62828（深红色）

动画效果：
1. 轻微震动（左右晃动）
2. 正确答案高亮闪烁
3. 修为数字显示"-X"或"不变"
```

### 部分正确/开放题评分
```
颜色方案：
- 背景：#FFF8E1（浅黄色）
- 边框：#FFC107（黄色）
- 图标：◐ 半圆图标
- 文字：#F57F17（深黄色）

动画效果：
1. 缩放弹出（较慢）
2. 评分进度条填充
3. 维度分数依次显示
```
