# 错题记录系统 (Wrong Question Record System)

> **Status**: Designed
> **Author**: User + Claude Code
> **Last Updated**: 2026-03-30
> **Implements Pillar**: 成长可见, 实用至上

## Overview

错题记录系统负责记录玩家答错的题目，将其转化为"心魔"概念，支持后续复习和攻克。

系统自动记录每次答错的题目ID、错误次数、首次答错时间、最近答错时间、最近复习时间。玩家可以在"心魔场景"中复习这些错题，答对后减少错误计数或移除心魔。心魔数量和难度影响"心魔压迫感"，增加复习紧迫性。

**为什么需要这个系统**: 没有错题记录，玩家无法知道自己哪些知识点薄弱，复习变成盲目刷题。系统化的错题记录让复习"有的放矢"，效率大幅提升。心魔概念将"错题"转化为修仙叙事元素，让枯燥的复习变成"攻克心魔"的仪式感。

## Player Fantasy

玩家期望体验：

**1. 知己知彼的安全感**
- "我知道自己哪里薄弱，不会盲目刷题"
- 每次答错，系统自动记录，玩家无需手动操作
- 心魔列表清晰展示所有待攻克的知识点

**2. 攻克障碍的成就感**
- 错题不再是"失败"，而是"心魔"——修仙路上的障碍
- 每次攻克心魔，都有仪式感反馈（心魔消散动画/音效）
- 心魔数量减少 = 实力提升的可见证明

**3. 持续进步的可见性**
- 心魔压迫感随数量增加，提醒玩家"该复习了"
- 攻克心魔后压迫感减弱，形成正向反馈循环
- 历史心魔记录展示成长轨迹（曾经攻克过多少心魔）

## Detailed Design

### Core Rules

#### 心魔记录触发
| 触发场景 | 操作 | 数据变更 |
|----------|------|----------|
| 普通修炼答错 | 新增或更新心魔 | `wrong_count += 1`, `last_wrong_at = now()` |
| BOSS面试答错 | 新增或更新心魔 | `wrong_count += 1`, `last_wrong_at = now()` |
| 心魔复习答对 | 减少错误计数 | `wrong_count -= 1`, `last_review_at = now()` |
| 心魔复习答错 | 保持或增加 | `wrong_count += 1`, `last_wrong_at = now()` |
| 错误计数归零 | 移除心魔 | 从 `wrong_questions` 中删除该 question_id |

#### 心魔数据结构
```gdscript
# 存储在 SaveData.wrong_questions 中
# 键: question_id (String)
# 值: Dictionary
{
    "wrong_count": int,        # 错误次数 (>=1)
    "first_wrong_at": String,  # 首次答错时间 ISO8601
    "last_wrong_at": String,   # 最近答错时间 ISO8601
    "last_review_at": String   # 最近复习时间 ISO8601 (可为空)
}
```

#### 心魔移除规则
- **默认**: `wrong_count` 降为 0 时自动移除
- **可选**: 提供"彻底斩除"选项，答对一次即移除（设置项）

#### 心魔压迫感
- 心魔数量越多，压迫感越强
- 压迫感影响：主界面视觉特效、心魔场景氛围
- 不影响数值，仅影响体验

### States and Transitions

```
┌─────────────────┐
│   无心魔状态     │  question_id 不在 wrong_questions 中
└────────┬────────┘
         │ 答错题目
         ↓
┌─────────────────┐
│   心魔初生       │  wrong_count = 1, first_wrong_at = now()
└────────┬────────┘
         │ 再次答错
         ↓
┌─────────────────┐
│   心魔加深       │  wrong_count += 1
└────────┬────────┘
         │ 心魔复习答对
         ↓
┌─────────────────┐
│   心魔削弱       │  wrong_count -= 1
└────────┬────────┘
         │ wrong_count = 0
         ↓
┌─────────────────┐
│   心魔斩除       │  从 wrong_questions 移除，触发斩除动画
└─────────────────┘
```

### Interactions with Other Systems

| 系统 | 数据流向 | 接口 |
|------|----------|------|
| **存档系统** | 双向 | `get/add/update/remove_wrong_question()` |
| **题库数据系统** | 读取 | `get_question_by_id(question_id)` 获取题目详情 |
| **心魔复习系统** | 读取+写入 | `get_wrong_questions()`, `update_after_review()` |
| **心魔UI系统** | 读取 | `get_wrong_questions()`, `get_pressure_level()` |
| **答题修炼系统** | 写入 | `add_wrong_question(question_id)` |
| **BOSS面试系统** | 写入 | `add_wrong_question(question_id)` |

## Formulas

### 心魔压迫感计算
```
pressure_level = min(wrong_question_count / pressure_threshold, 1.0)

where:
  wrong_question_count = 心魔总数
  pressure_threshold = 触发满压迫感的心魔数量（默认20）
```

**示例计算**:
- 5个心魔: `5 / 20 = 0.25` → 压迫感25%
- 10个心魔: `10 / 20 = 0.5` → 压迫感50%
- 20个心魔: `20 / 20 = 1.0` → 压迫感100%（满）

### 心魔斩除条件
```
can_exorcise = (review_correct_count >= required_correct_count)

where:
  review_correct_count = 连续答对次数（复习模式下）
  required_correct_count = 需要连续答对的次数（默认2）
```

**设计意图**: 单次答对可能是侥幸，连续答对证明真正掌握。

### 心魔危险度评级
```
danger_level = min(wrong_count, 3)

where:
  wrong_count = 累计答错次数
  danger_level ∈ {1, 2, 3} 对应 {低, 中, 高}
```

**示例**:
- 答错1次 → 低危心魔
- 答错2次 → 中危心魔
- 答错3次及以上 → 高危心魔

### 心魔复习优先级排序
```
priority_score = wrong_count × 10 + days_since_last_review × 2

where:
  wrong_count = 累计答错次数
  days_since_last_review = 距上次复习的天数
```

**排序规则**: priority_score 越高，优先级越高（越应该先复习）。

**示例**:
- 心魔A: 答错3次，3天没复习 → `3×10 + 3×2 = 36`
- 心魔B: 答错1次，10天没复习 → `1×10 + 10×2 = 30`
- 心魔C: 答错2次，7天没复习 → `2×10 + 7×2 = 34`
- 排序: A > C > B

## Edge Cases

| 边缘情况 | 处理方式 | 影响范围 |
|----------|----------|----------|
| **首次答错** | 创建心魔记录，设置 `first_wrong_at` 和 `last_wrong_at` 为当前时间，`wrong_count = 1`，`last_review_at` 为空 | 答题流程 |
| **心魔复习答错同一题** | `wrong_count += 1`，复习失败，心魔加深 | 心魔复习系统 |
| **心魔复习答对但 wrong_count > 1** | `wrong_count -= 1`，心魔削弱但不移除，需继续复习 | 心魔复习系统 |
| **同一题目反复答错多次** | `wrong_count` 累加，危险度升级，优先级提高 | 答题/复习流程 |
| **心魔复习答对两次但中间答错** | 计数重置，不触发斩除，需重新连续答对 | 心魔复习系统 |
| **心魔列表为空** | 心魔场景显示"暂无心魔"提示，压迫感为0 | 心魔UI系统 |
| **题目被从题库删除** | 心魔记录保留，复习时跳过该题，显示"已失效"标记 | 题库维护 |
| **存档损坏导致心魔数据丢失** | 心魔列表为空，玩家需重新答题触发心魔 | 存档系统 |
| **心魔复习中途退出** | 当前复习进度不保存，下次从心魔列表重新选择 | 心魔复习系统 |
| **达到心魔上限（极端情况）** | 仍可继续记录，压迫感保持100%，显示警告提示玩家复习 | 答题流程 |

## Tuning Knobs

| Knob | 类型 | 默认值 | 范围 | 影响范围 |
|------|------|--------|------|----------|
| `pressure_threshold` | int | 20 | 10-50 | 触发满压迫感的心魔数量 |
| `required_correct_count` | int | 2 | 1-5 | 斩除心魔需要连续答对的次数 |
| `max_wrong_count_display` | int | 3 | 3-10 | 心魔危险度显示上限（超过仍累计） |
| `review_priority_wrong_weight` | int | 10 | 1-20 | 优先级公式中 wrong_count 的权重 |
| `review_priority_days_weight` | int | 2 | 1-10 | 优先级公式中天数的权重 |
| `pressure_warning_threshold` | float | 0.7 | 0.5-0.9 | 触发"心魔过多"警告的压迫感阈值 |

### 调参指南

**`pressure_threshold`（压迫感阈值）**
- **调低**: 玩家更快感受到紧迫感，适合硬核玩家
- **调高**: 玩家更从容，适合休闲玩家
- **观察指标**: 玩家平均心魔数量、复习频率

**`required_correct_count`（斩除所需答对次数）**
- **调低**: 更容易斩除心魔，正向反馈更快
- **调高**: 斩除更困难，确保真正掌握
- **观察指标**: 心魔平均存活时间、重复犯错率

**`pressure_warning_threshold`（警告阈值）**
- **调低**: 更早提醒玩家复习
- **调高**: 减少打扰，让玩家自主决定
- **观察指标**: 玩家对警告的反应（是否真的去复习）
