# 答题修炼系统 (Practice Cultivation System)

> **Status**: Designed
> **Author**: User + Claude Code
> **Last Updated**: 2026-03-30
> **Implements Pillar**: 实用至上, 成长可见

## Overview

**答题修炼系统**负责管理玩家日常答题修炼的完整流程，包括题目抽取、答题交互、答案判定、修为奖励、心魔记录。

这是游戏的核心玩法系统，玩家通过答题获得修为、提升境界、攻克心魔。

**为什么需要这个系统**: 这是游戏的核心循环——答题→修为→境界。没有这个系统，游戏没有玩法内容。

## Player Fantasy

玩家期望体验：

**1. 刷题的成就感**
- 每道题都有收获（修为+知识点）
- 看到修为增长、进度条前进
- 攻克心魔后的成就感

**2. 学习的实用性**
- AI 反馈帮助理解知识点
- 错题自动记录，复习有方向
- 面试真题，学了就能用

**3. 自由的选择权**
- 自选题目类型/难度
- 随时可停止，碎片友好
- 不强制顺序，按兴趣学习

## Detailed Design

### Core Rules

#### 题目抽取规则

| 抽取方式 | 规则 | 用途 |
|----------|------|------|
| **随机抽取** | 从题库随机抽取 | 快速刷题 |
| **按难度抽取** | 指定难度范围 | 针对性练习 |
| **按知识点抽取** | 指定标签/领域 | 专项突破 |
| **心魔复习** | 从心魔列表抽取 | 攻克心魔 |

#### 答题流程

```
1. 玩家选择题目（或系统随机抽取）
2. 显示题目界面
   - 题面
   - 选项/代码编辑器
   - 计时器（可选）
3. 玩家作答
4. 提交答案
5. 调用回答分析系统判定
6. 显示答题反馈
7. 更新数据
   IF 正确:
     - 增加修为
     - 记录答题正确
   ELSE:
     - 记录心魔
     - 不扣除修为
8. 返回修炼场景或继续答题
```

#### 修为奖励

| 题目类型 | 难度 | 基础修为 | 标签加成 |
|----------|------|----------|----------|
| 选择题 | 简单 | 10 | +10%/标签 |
| 选择题 | 中等 | 20 | +10%/标签 |
| 选择题 | 困难 | 30 | +10%/标签 |
| 代码题 | 简单 | 15 | +10%/标签 |
| 代码题 | 中等 | 25 | +10%/标签 |
| 代码题 | 困难 | 35 | +10%/标签 |
| 开放题 | - | 按评分 | 见评分机制 |

## Formulas

### 修为奖励计算
```
cultivation_gain = base + (base × tag_bonus)

where:
  base = 题目难度对应的基础修为
  tag_bonus = min(tag_count × 0.1, 0.5)  // 最高50%加成
```

**示例**:
- 中等选择题，标签3个 → `20 + (20 × 0.3) = 26`
- 困难代码题，标签5个 → `35 + (35 × 0.5) = 52.5`

### 连续答题奖励
```
streak_bonus = min(consecutive_correct × 0.05, 0.3)

where:
  consecutive_correct = 连续答对数量
```

**示例**:
- 连续答对3题 → `+15%` 修为
- 连续答对6题 → `+30%` 修为（上限）

## Edge Cases

| 边缘情况 | 处理方式 |
|----------|----------|
| **题库题目已答完** | 提示"已答完所有题目"，允许重复 |
| **答题超时** | 按答错处理，记录心魔 |
| **答题中途退出** | 不记录答题，返回场景 |
| **修为达到上限** | 提示"修为已满，请突破" |
| **心魔数量达到上限** | 提示"心魔过多，建议复习" |
