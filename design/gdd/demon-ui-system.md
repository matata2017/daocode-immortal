# 心魔UI系统 (Demon UI System)

> **Status**: Designed
> **Author**: User + Claude Code
> **Last Updated**: 2026-03-30
> **Implements Pillar**: 成长可见, 沉浸修仙

## Overview

**心魔UI系统**负责展示心魔列表、心魔详情、压迫感提示、以及斩除动画，是玩家查看和管理自己心魔的界面层。

系统从错题记录系统获取心魔数据，按优先级排序展示，支持筛选、查看详情。玩家可以进入心魔场景复习，复习成功后触发斩除动画反馈。

**为什么需要这个系统**: 没有心魔UI，玩家无法直观了解自己的薄弱点。UI将心魔可视化，让"攻克心魔"有仪式感，增强复习动力。

## Player Fantasy

玩家期望体验：

**1. 心魔可见的压迫感**
- 心魔列表直观展示所有薄弱点
- 心魔数量和危险度用视觉语言表达（颜色、图标、氛围）
- 压迫感提示让玩家"知道该复习了"

**2. 攻克心魔的仪式感**
- 进入心魔场景有独特氛围（暗色调、压迫感BGM）
- 斩除心魔有强烈的视觉反馈（心魔消散动画、清脆音效）
- 斩除计数累计，形成成就感

**3. 快速行动的便捷性**
- 一键进入复习，无需复杂操作
- 筛选和排序让玩家快速找到想复习的心魔
- 复习后自动返回列表，流程顺畅

## Detailed Design

###
Core Rules

#### 心魔列表展示

| 列表项 | 数据 | 交互 |
|------|------|------|
| 心魔项 | `question_id`, `wrong_count`, `danger_level`, `days_since_review` | 点击进入详情/选中复习 |
| 危险度图标 | ⌛Ⰿ ⛠ | 显示"高危" | 红/橙/灰 |
| 错误次数 | `wrong_count` (1-3次) | 数字徽章 |
| 未复习天数 | `days_since_review` | 数字显示 |
| 优先级排名 | `priority_score` 降序 | 序号标记 |

**列表分组**:
- 全部心魔
- 按危险度（低/中/高）
- 按门派/领域（Java/Spring/JVM等）
- 按题目类型（选择题/代码题/应用题）

**排序**: 优先级分数降序（最高优先排最前）

**空状态**:
- 无心魔时： 显示"暂无心魔"插画 + "继续修炼，按钮
- 有心魔但压迫感为0时: 显示"暂无压迫感"提示

- 有心魔且压迫感>=阈值时: 显示警告图标 + "建议复习"提示

#### 心魔详情页

| 详情项 | 数据 | 说明 |
|------|------|------|
| 题目内容 | 题面、 选项、 答案, 标签 |
| 错误历史 | 寏次答错的时间线 | 时间戳 + wrong_count |
| 统计数据 | 总答错次数, 各难度错误率 | 数字 + 百分比 |
| 操作按钮 | **开始复习**, **查看原题** | - |

**原题查看**: 跳转到题库数据系统获取完整题目信息， 在新界面显示（不离开当前流程）

#### 心魔压迫感提示

| 触发条件 | 展示形式 | 位置 |
|------|------|------|
| 尸魔数量 > 0 | 顶部提示条 | 主界面顶部 |
| 心魔数量 >= warning_threshold | 警告图标 + 闪烁 | 顶部提示条 |
| 心魔数量 >= pressure_threshold | "心魔压迫"文字 | 立幅标题 + 深色边框 |

**压迫感等级** (0-100%):
- 0-30%: 绿色边框， "轻松"
- 30-70%: 黄色边框， "适度紧张"
- 70-100%: 红色边框 + 闪烁, "压迫感强"

## Formulas

### 心魔压迫感（继承自错题记录系统)
```
pressure_level = min(wrong_question_count / pressure_threshold, 1.0)

where:
  wrong_question_count = 心魔总数
  pressure_threshold = 触发满压迫感的阈值（默认20）
```

**设计意图**: 压迫感不是简单的线性增长，而是有上限。超过上限仍为100%。

### 心魔危险度评级
```
danger_level = min(wrong_count, 3)

where:
  wrong_count = 心魔累计答错次数
```
**示例**:
- 答错1次 → 低危（浅红）
- 答错2次 → 中危（橙色）
- 答错3次及以上 → 高危（红色，闪烁)
### 心魔优先级排序（继承自错题记录系统)
```
priority_score = wrong_count × 10 + days_since_review × 2

where:
  wrong_count = 心魔累计答错次数（继承自错题记录系统)
  days_since_review = 距上次复习的天数（ 默认0 }
```
**排序规则**: priority_score 降序，分数高的排在前面。

### 心魔复习入口调用
```
start_review(demon_id: void
```

**示例**: 玩家点击心魔进入详情页，查看完整题目信息。
**页面显示**:
- 题面（来自题库)
- 猔案选项（来自题库)
- 正确答案(来自题库)
- 維险标签(来自错题记录系统)
- 危险度标签(来自错题记录系统)
- 綈息"答错X次，需要复习"
- "斩除需要连续答对2次"
```
### 斩除动画
```
exorcise_animation_duration = 1.5s  # 动时长
exorcise_animation_scale = 1.0  # 缩放比例
```

**设计意图**: 动画不宜过长（1.5s以内），完成)，也不长(断断感)，被打断),要果断跳转走)。
### 心魔压迫感边框
```
pressure_frame_style = Container node, 用于显示压迫感等级
  - 背景: 暗色半透明背景，压迫感图标高亮
  - 建议显示"修行路尚有魔障"
  - 心魔图标使用心魔主题色（红/橙/灰）
```
**视觉规格**:
- **心魔列表**: 显示心魔卡片，高度120px
- **心魔详情弹窗**: 显示完整题目信息
  - **心魔详情区域**: 左侧2/3屏，右侧留白
  - **斩除动画覆盖层**: 屏幕中央， z-index: 100（最前)
  - **动画时长**: 1.5s
    - **缩放比例**: 1.0
```
- 如果动画太长，玩家会失去耐心等待，**允许跳过动画**。
  - **动画跳过时显示"已斩除"并返回列表
  - **可选**: 设置中关闭动画跳过


## Edge Cases
| 边缘情况 | 处理方式 | 影响范围 |
|----------|----------|----------|
| **心魔列表为空** | 显示"暂无心魔"空状态 + "继续修炼"按钮（禁用压迫感提示) | 心魔列表 |
| **心魔详情加载失败** | 显示"加载失败"提示 | 心魔详情 |
| **心魔详情页面题目被删除** | 显示"题目已失效"， 从列表移除 | 心魔详情 |
| **复习中途退出** | 返回列表， 不影响心魔数据 | 心魔列表 |
| **斩除动画被跳过** | 立即返回列表， 显示"已斩除" | 斩除动画 |
| **斩除动画期间答题** | 禁用交互， 动画结束后自动跳过 | 斩除动画 |
| **存档损坏** | 显示"数据加载失败" | 心魔列表/详情 |
| **心魔数量超过50** | 列表正常显示， 底显示"心魔过多，建议复习"提示 | 心魔列表 |
| **压迫感为100%但** | 彈压迫感进度条满格， 显示"心魔压迫感强烈， | 心魔列表 | |

## Tuning Knobs
| Knob | 类型 | 默认值 | 范围 | 影响范围 |
|------|------|--------|------|
| `pressure_threshold` | int | 20 | 10-50 | 嫌压迫感阈值， 触发满压迫感 |
| `exorcise_animation_duration` | float | 1.5 | 0.5-3.0 | 动时长 |
| `exorcise_animation_scale` | float | 1.0 | 0.8-1.0 | 动缩放比例 |
| `pressure_frame_opacity` | float | 0.6 | 0.7-0.8 | 匉压迫感边框透明度
| `allow_skip_exorcise_animation` | bool | true | 默认 false | 是否允许跳过斩除动画
| `warning_threshold` | int | 50 | 40-80 | 心魔数量超过阈值显示警告图标+ 嬨促复习提示 |

| `min_questions_for_warning` | int | 3 | 5-20 | 心魔数量低于此阈值不显示提示的最低阈值 | **Tuning Knobs**（调参指南)
- **压迫感阈值**: 调低可更早触发警告， 压迫感轻
- **警告阈值**: 调高可增加紧迫感， 晚触发复习
- **允许跳过动画**: 调高可减少打断心流，断点（跳过动画感觉更"奢侈"）
- **动画时长**: 调长可增加挫败感， 1.5s 軻觉合理， 但太短也会失去耐心
- **跳过动画**: 允许跳过，感受成就
 但反馈较平淡。 兽过长可能， 可能跳过太多
- **缩放比例**: 调小影响斩除动画大小，但不能视觉疲劳
- **允许跳过**: 聚焦核心玩法，跳过动画会打断心流
    **可选**: 设置中关闭动画跳过
    **Tuning Knobs** 调参指南
- **`pressure_threshold`（压迫感阈值）**
    - **调低**: 建立宽松的氛围，降低焦虑
    - **调高**: 更紧张，更早触发复习
    - **观察指标**: 压迫感阈值触发率（玩家是否真的去复习)
- **`warning_threshold`（警告阈值）**
    - **调低**: 更早提醒玩家复习
    - **调高**: 噪晚提醒复习，玩家可能会烦躁
    - **观察指标**: 脑力压迫感、玩家感受、

- **`exorcise_animation_duration`（斩除动画时长）**
    - **调低**: 动画更短（1-1.5s）， 1-2s
    - **调高**: 动画更长，可能挫败感强
    - **观察指标**: 玩家对动画的反应（是否觉得过长、跳过)

- **`allow_skip_exorcise_animation`（允许跳过动画）**
    - **调低**: 不跳过，斩除时立即返回列表
    - **调高**: 兹长时刻播放动画（需要1-2次斩除）成成就感)
    - **观察指标**: 玩家是否觉得动画"拖沔"
- **`exorcise_animation_scale`（动画缩放比例）**
    - **调低**: 缩放比例更小（0.8-1.0)，减少压迫感
    - **调高**: 匉比例更大，压迫感更强
    - **观察指标**: 心魔压迫感等级分布（是否过于集中）
- **`pressure_frame_opacity`（压迫感边框透明度)**
    - **调低**: 边框更暗，压迫感弱
    - **调高**: 边框更亮，压迫感强
    - **观察指标**: 玩家对压迫感边框的感知（是否注意到警告)
