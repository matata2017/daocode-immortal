# 突破考试系统 (Breakthrough Exam System)

> **Status**: In Design
> **Author**: User + Claude Code
> **Last Updated**: 2026-03-30
> **Implements Pillar**: 沉浸修仙, 成长可见

## Overview

**突破考试系统**负责管理玩家从当前境界突破到下一境界的考试流程，包括考试触发条件、题目抽取、答题流程、判定规则、突破成功/失败处理。

系统在玩家修为达到阈值时触发突破考试，从题库抽取指定数量题目，玩家答题后根据正确率判定是否突破成功。成功后境界提升，失败后冷却时间等待重试。

**为什么需要这个系统**: 没有突破考试，境界提升只是"修为够了就升级"，缺乏仪式感和挑战性。系统化的突破考试让境界提升有"仪式感"，考试压力增加沉浸体验，正确率要求确保玩家真正掌握知识点。

## Player Fantasy

玩家期望体验：

**1. 突破的仪式感**
- "我要突破到下一境界了！"的期待感
- 专属突破场景（天劫、心魔试炼等）
- 专属突破BGM（激昂、振奋）
- 突破成功时的庆祝特效

**2. 考试的压力感**
- 限时答题增加紧张感
- 正确率门槛让每道题都重要
- 失败后的冷却期增加珍惜感

**3. 成长的里程碑感**
- 境界名称变化（练气→筑基→金丹...）
- 境界图标/颜色变化
- 解锁新能力/新题目

## Detailed Design

### Core Rules

#### 突破条件

| 境界 | 修为阈值 | 考试题目数 | 正确率要求 | 失败冷却 |
|------|----------|------------|------------|----------|
| 练气期→筑基期 | 100 | 5题 | 60% | 1小时 |
| 筑基期→金丹期 | 500 | 8题 | 65% | 2小时 |
| 金丹期→元婴期 | 1500 | 10题 | 70% | 4小时 |
| 元婴期→化神期 | 4000 | 12题 | 75% | 8小时 |
| 化神期→渡劫期 | 10000 | 15题 | 80% | 24小时 |

#### 考试流程

```
1. 触发突破考试
   - 玩家修为 >= 境界阈值
   - 玩家点击"突破"按钮

2. 检查条件
   - 修为是否达标
   - 是否在冷却期内
   - 前置境界是否已达成

3. 进入突破场景
   - 播放突破BGM
   - 显示突破提示

4. 抽取考试题目
   - 从题库抽取指定数量题目
   - 题目难度匹配目标境界
   - 随机打乱顺序

5. 答题流程
   - 逐题显示
   - 玩家作答
   - 答题反馈

6. 计算正确率
   - correct_count / total_count

7. 判定结果
   IF correct_rate >= threshold:
     突破成功
     - 播放成功动画
     - 更新境界
     - 解锁新内容
   ELSE:
     突破失败
     - 播放失败提示
     - 设置冷却时间
     - 返回修炼场景
```

#### 题目抽取规则

```
抽取规则:
1. 题目难度分布：
   - 基础题（当前境界）: 30%
   - 进阶题（目标境界）: 50%
   - 挑战题（跨境界）: 20%

2. 知识点覆盖：
   - 从不同领域抽取
   - 避免重复知识点

3. 题型比例：
   - 选择题: 60%
   - 代码题: 30%
   - 应用题: 10%
```

## Formulas

### 突破判定
```
can_breakthrough = (
    cultivation >= cultivation_threshold AND
    cooldown_remaining <= 0 AND
    current_realm == required_previous_realm
)

breakthrough_success = correct_count / total_count >= accuracy_threshold
```

**示例**:
- 筑基期→金丹期： 修为>=500, 正确率>=65%
- 答对5/8题 → `5/8 = 62.5% < 65%` → 失败
- 答对6/8题 → `6/8 = 75% >= 65%` → 成功

### 冷却时间计算
```
cooldown_remaining = max(0, cooldown_end_time - current_time)

where:
  cooldown_end_time = failure_time + cooldown_duration
  cooldown_duration = 境界对应的冷却时长
```

### 题目抽取
```
questions = sample_questions(
    pool = all_questions,
    difficulty_distribution = {basic: 0.3, advanced: 0.5, challenge: 0.2},
    count = exam_question_count,
    seed = player_id + current_timestamp
)
```

### 突破失败惩罚
```
IF breakthrough_failed:
    cooldown_start = now()
    cultivation_loss = cultivation × 0.1  // 扣除10%修为
    new_cultivation = cultivation - cultivation_loss
```

**设计意图**: 失败有代价，但不至于让玩家前功尽弃。10%修为是可接受的损失。

## Edge Cases

| 辝缘情况 | 处理方式 | 影响范围 |
|----------|----------|----------|
| **考试中途退出** | 记录已答题数，本次失败， 设置冷却 | 考试流程 |
| **题库题目不足** | 使用已有题目 + 提示"题库不足" | 考试流程 |
| **考试过程中网络断开** | 暂停计时，允许重连继续 | 网络问题 |
| **答对后立即关闭游戏** | 保存进度， 下次继续 | 存档系统 |
| **突破成功时播放动画失败** | 跳过动画， 显示成功文字 | 动画系统 |
| **已满级玩家重复突破** | 提示"已达最高境界" | 境界系统 |
| **冷却期内点击突破** | 显示剩余冷却时间， 禁止突破 | UI提示 |
| **修为刚好等于阈值** | 允许突破（>=也 也可触发） | 突破判定 |
| **考试中答对所有题目** | 仍计算正确率， 判定结果 | 考试流程 |

## Tuning Knobs

| Knob | 类型 | 默认值 | 范围 | 影响范围 |
|------|------|--------|------|----------|
| `base_cooldown_hours` | int | 1 | 0-24 | 基础冷却时间（小时） |
| `failure_cultivation_loss_rate` | float | 0.1 | 0.0-0.3 | 失败扣除修为比例 |
| `accuracy_threshold` | Dictionary | 见境界表 | 50%-90% | 各境界正确率要求 |
| `exam_time_limit_per_question` | int | 60 | 30-120 | 每题答题时限（秒） |
| `show_correct_answer_on_fail` | bool | true | true/false | 答错时是否显示正确答案 |
| `allow_skip_exam_animation` | bool | true | true/false | 是否允许跳过突破动画 |

### 调参指南

**`failure_cultivation_loss_rate`（失败修为损失）**
- **调低**: 失败代价小，玩家更愿意尝试
- **调高**: 失败代价大，玩家更谨慎准备
- **观察指标**: 突破尝试次数 vs 成功率

**`accuracy_threshold`（正确率要求）**
- **调低**: 更容易突破，节奏更快
- **调高**: 更难突破，成就感更强
- **观察指标**: 突破成功率、 玩家挫败感反馈

**`exam_time_limit_per_question`（答题时限）**
- **调低**: 增加紧张感，考验快速思考
- **调高**: 给予更多思考时间，减少焦虑
- **观察指标**: 超时率、 答题正确率
