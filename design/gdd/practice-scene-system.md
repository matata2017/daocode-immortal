# 修炼场景系统 (Practice Scene System)

> **Status**: Designed
> **Author**: User + Claude Code
> **Last Updated**: 2026-03-30
> **Implements Pillar**: 沉浸修仙, 碎片友好

## Overview

**修炼场景系统**负责管理玩家日常修炼的场景，包括场景加载、氛围渲染、答题界面展示、修为进度显示。

修炼场景是玩家最常驻的地方，需要营造"专注修炼"的氛围，同时支持快速进入答题流程。

**为什么需要这个系统**: 没有修炼场景，玩家答题没有"地点感"，只是单纯的答题界面。系统化的场景让修炼有"仪式感"，增强沉浸体验。

## Player Fantasy

玩家期望体验：

**1. 专注的修炼氛围**
- 安静、平和的BGM
- 简洁的UI，不分散注意力
- 修为进度清晰可见

**2. 快速进入答题**
- 一键开始修炼（答题）
- 无复杂流程，即点即答
- 支持碎片时间快速刷题

**3. 个性化场景**
- 不同门派有不同修炼场景
- 境界提升场景有变化
- 场景有动态元素（粒子、光影）

## Detailed Design

### Core Rules

#### 场景类型

| 场景 | 门派 | 境界 | BGM | 特效 |
|------|------|------|-----|------|
| 初级修炼室 | 所有门派 | 练气-筑基 | `practice_basic.mp3` | 轻微粒子 |
| 中级修炼室 | 所有门派 | 金丹-元婴 | `practice_medium.mp3` | 中等粒子+光影 |
| 高级修炼室 | 所有门派 | 化神-渡劫 | `practice_advanced.mp3` | 重粒子+光环 |
| 门派专属 | 各门派 | 所有境界 | 门派BGM | 门派特色 |

### States and Transitions

```
┌─────────────────┐
│   场景加载中     │  加载资源
└────────┬────────┘
         │ 加载完成
         ↓
┌─────────────────┐
│   修炼场景展示   │  显示场景UI
└────────┬────────┘
         │ 点击开始修炼
         ↓
┌─────────────────┐
│   进入答题流程   │  调用答题修炼系统
└────────┬────────┘
         │ 答题完成
         ↓
┌─────────────────┐
│   返回修炼场景   │  更新修为显示
└─────────────────┘
```

### Interactions with Other Systems

| 系统 | 数据流向 | 接口 |
|------|----------|------|
| **场景管理系统** | 读取 | `load_scene("practice")` |
| **音频系统** | 写入 | `play_bgm("practice")` |
| **答题修炼系统** | 写入 | `start_practice()` |
| **修为计算系统** | 读取 | `get_cultivation_progress()` |
| **门派数据系统** | 读取 | `get_faction_theme()` |

## Formulas

### 场景选择逻辑
```
practice_scene = get_scene_for_realm(current_realm)

where:
  练气-筑基 → 初级修炼室
  金丹-元婴 → 中级修炼室
  化神-渡劫 → 高级修炼室
```

### 修为进度显示
```
progress_percent = current_cultivation / next_threshold × 100

where:
  current_cultivation = 当前修为
  next_threshold = 下一境界阈值
```

## Edge Cases

| 边缘情况 | 处理方式 |
|----------|----------|
| **已达最高境界** | 显示"已圆满"，不显示进度条 |
| **场景加载失败** | 显示错误，使用默认场景 |
| **BGM加载失败** | 静默继续 |
