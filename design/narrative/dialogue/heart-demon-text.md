# 心魔对话模板

> **状态**: 最终版
> **作者**: Claude Code (文案)
> **创建时间**: 2026-03-30
> **版本**: 1.0

---

## 概述

心魔是玩家错误的叙事具象化。它们不是要对抗的敌人，而是要承认和克服的弱点。它们的对话设计为挑战但不打击，刺激但不挫败。

---

## 对话类别

### 1. 认知模板

**目的**: 承认过去的错误并邀请玩家面对它。

**何时使用**: 首次遭遇特定主题的心魔。

```
# 主要

"记得这个吗？你之前不理解 {TOPIC}。
现在呢？"

# 替代

"{TOPIC} 曾经绊倒过你。还会吗？"

"你的心智仍带着 {TOPIC} 的阴影。
该面对它了。"

"这问题曾经击败过你。你的修为有没有增长？"

"{TOPIC}...你理解中的黑点。面对它。"

"{TOPIC} 失败的记忆还在。面对它，成长。"

"你曾在这里跌倒。{TOPIC} 仍有秘密。"

"{TOPIC} 的阴影还在。你会驱散它吗？"

"你过去的自己无法回答这个。现在的你能吗？"

# 随和变体

"嘿，记得 {TOPIC} 吗？是的，那是个难题。
让我看看你现在有没有掌握了。"

# 正式变体

"一个心魔从你在 {TOPIC} 的错误中显现。
承认它。克服它。"
```

### 2. 怀疑模板

**目的**: 质疑玩家的准备程度并引发自我反思。

**何时使用**: 玩家在同一问题上多次失败，或在心魔复习中答错。

```
# 主要

"{TOPIC} 仍然让你困惑。你确定准备好了吗？"

# 替代

"你在 {TOPIC} 上的基础还很薄弱。是不是应该先巩固一下？"

"即使现在，{TOPIC} 仍让你困惑。或许需要更多修炼。"

"你再次面对我，但你的理解没有加深。"

"{TOPIC} 在你心中仍模糊。阴影持续。"

"这不是 {TOPIC} 第一次击败你。"

"同样的错误。同样的黑暗。你永远学不会吗？"

"提到 {TOPIC} 时你的真气就动摇。为何？"

# 温和怀疑

"看来 {TOPIC} 对你还是困难。
没关系。理解需要时间。"

"你在 {TOPIC} 上挣扎。或许面对这个心魔前先复习一下？"

# 锐利怀疑

"你的修为真的足够应对 {TOPIC} 吗？
阴影怀疑。"

"{TOPIC} 已经击败你多次。这次有何不同？"
```

### 3. 挑战模板

**目的**: 邀请玩家证明他们的成长。

**何时使用**: 当玩家主动面对心魔，或作为鼓励再试一次。

```
# 主要

"证明你已掌握 {TOPIC}。驱散这个阴影。"

# 替代

"让我看看你的修为是否提升。战胜这个弱点。"

"展示你对 {TOPIC} 的理解。现在。"

"如果你真的理解 {TOPIC}，这应该简单。是吗？"

"前进的路很清楚。答对，我就消失。"

"你的成长在像这样的时刻衡量。证明你自己。"

"你要让 {TOPIC} 永远纠缠你吗？还是你要掌握它？"

"这个阴影存在因为你曾经失败。让它不再重要。"

# 鼓励型挑战

"你从那时起修炼了。向这个心魔展示你的成长。"

"你现在知道更多。向你自己证明。"

"答案在你体内。找到它。"

# 直接挑战

"回答。正确。然后摆脱 {TOPIC}。"

"面对我。击败我。前进。"
```

### 4. 拒绝模板

**目的**: 在反复失败后嘲讽玩家（但仍留有成长空间）。

**何时使用**: 当玩家在心魔复习中答错。

**设计说明**: 拒绝应当尖锐但不残酷。玩家必须感到他们能克服。

```
# 主要

"{TOPIC} 仍是你的弱点。阴影更强了。"

# 替代

"你对 {TOPIC} 的理解仍不完整。
修炼更多后回来。"

"你没有成长。这个心魔还在。"

"阴影以你的失败为食。{TOPIC} 再次击败你。"

"还被 {TOPIC} 困惑？或许需要更多学习。"

"你的修为对 {TOPIC} 还不够。
接受这个。思考它。更强后回来。"

"同样的黑暗。同样的错误。{TOPIC} 再次吞噬你。"

"你来面对弱点，却向它投降了。"

# 温和拒绝

"尚未。{TOPIC} 需要更多修炼。
学习它。理解它。然后回来。"

"这个心魔还没准备好消散。
你的理解需要深度。去。学习。回来。"

# 锐利拒绝

"你以为你准备好了。{TOPIC} 的阴影说不是。"

"没有理解的思考毫无成就。
{TOPIC} 会等你回来。你会准备好吗？"

# 恢复钩子

"但阴影终会消散。{TOPIC} 可以被掌握。
多修炼。准备好了回来。"
```

### 5. 解脱模板

**目的**: 当玩家成功驱散心魔时承认他们的成长。

**何时使用**: 当玩家正确回答心魔问题，清除该主题。

```
# 主要

"{TOPIC}...你现在理解它了。这个心魔已驱散。"

# 替代

"你的修为已经提升。{TOPIC} 的阴影在消散。"

"你成长了。这个弱点...不再是弱点。"

"{TOPIC} 不再纠缠你。心魔化光消散。"

"你的理解加深了。{TOPIC} 的阴影消失了。"

"所以。你已掌握 {TOPIC}。心魔在这里没有力量。"

"曾有困惑之处，现在清明。
{TOPIC} 已征服。"

"黑暗升起。{TOPIC} 是你的了。这个心魔被击败。"

"你的真气不再在 {TOPIC} 处动摇。阴影...淡去。"

# 温暖解脱

"干得好。{TOPIC} 曾困扰你。不再了。"

"你来了，面对了，征服了。
{TOPIC} 现在在你身后。"

# 正式解脱

"凭你对 {TOPIC} 的理解，这个心魔被驱逐。
继续修炼，摆脱这个负担。"

# 成长承认

"记得 {TOPIC} 似乎不可能的时候吗？
看看你走了多远。"

"你一直在默默修炼。这看得出来。
{TOPIC} 不再对你有力量。"

# 最终消散

"{TOPIC} 的心魔化光消散。
你的心智更清明。你的道路更轻盈。

继续修炼。这个弱点已治愈。"
```

## 多条目心魔

当一个主题有多条错误条目时，使用进阶模板。

### 进阶模板

```
# 第一条目

"一个阴影从你在 {TOPIC} 的错误中显现。
一个错误。一个成长机会。"

# 第二条目

"{TOPIC} 的阴影更暗了。
两次失败。两课等着学习。"

# 第三条目

"你在 {TOPIC} 的困惑加深了。
三个错误。心魔以重复的错误为食。"

# 多条目（5+）

"{TOPIC} 周围的黑暗很浓。
许多失败累积。但每个阴影都可以驱散。"

# 鼓励进展

"你已清除 {TOPIC} 中 {CLEARED_COUNT} / {TOTAL_COUNT} 个阴影。
继续。理解随每次胜利增长。"

# 即将清完

"{TOPIC} 还剩最后一个阴影。
完成它。彻底清空你的心智。"
```

## 语境修饰符

### 基于玩家境界

```
# 低境界（炼气期、筑基期）

"你在 {TOPIC} 上的根基需要加固。
这对早期修士来说很常见。"

# 中境界（金丹期）

"即使在你这个境界，{TOPIC} 仍挑战你。
这不丢人。掌握它，变得更强。"

# 高境界（元婴期以上）

"你这个层次的修士，被 {TOPIC} 困扰？
即使大师也有缺口。填补这个。"
```

### 基于距答错的时间

```
# 近期（同一会话）

"{TOPIC} 的错误在你记忆中还很新。
趁教训清晰时现在面对它。"

# 中等（1-7 天）

"{TOPIC} 在你心中徘徊了好几天。
该面对它了。"

# 旧（7+ 天）

"{TOPIC} 的阴影等了很久。
但阴影不会永远等待。现在面对它。"
```

### 基于主题类别

```
# 算法

"{TOPIC} 的逻辑曾经逃过你。
你的推理有进步吗？"

# 数据结构

"{TOPIC} 的结构曾让你困惑。
你能看清它的形态吗？"

# 并发

"{TOPIC} 的并行思绪曾在你心中纠缠。
找到理解的线头。"

# 系统设计

"{TOPIC} 的架构曾超出你。
你有没有建设你的理解？"

# 语言特定

"{TOPIC} 的细微之处绊倒过你。
你的修炼有没有加深？"
```

## 快速参考

### 类别总结

| 类别 | 目的 | 语调 |
|----------|---------|------|
| 认知 | 承认过去错误 | 中性，邀请 |
| 怀疑 | 质疑准备程度 | 挑战，刺激 |
| 挑战 | 邀请证明成长 | 鼓励，直接 |
| 拒绝 | 承认持续失败 | 锐利，但不残酷 |
| 解脱 | 庆祝克服弱点 | 温暖，肯定 |

### 行长度指南

- 每行最多 120 字符
- 大多数行应为 50-80 字符
- 简短行（少于 40 字符）用于强调时刻

### 选择算法

1. 检查语境修饰符（境界、时间、主题）
2. 如有多条目则应用进阶模板
3. 从适当类别池中随机选择
4. 在同一会话中避免重复同一行
5. 匹配强度与情境（险败 = 更强烈）
