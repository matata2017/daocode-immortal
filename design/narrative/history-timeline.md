## 概述

本文档为修炼隐喻世界提供历史背景文字。**历史仅作为背景氛围——它不是叙事焦点。** 时间线将编程语言历史映射到修炼隐喻以提供风味，而非创造复杂的世界历史。

---

## 正典等级：临时

所有历史内容标记为 **临时**，允许在预制作期间演进。

---

## 历史设计原则

**原则**：历史仅是背景文字，以 2-3 句片段呈现。

- 没有复杂的各种事件时间线
- 没有日期或年份（使用相对时间："古代"、"现代"）
- 除 BOSS 记忆碎片外没有历史人物
- 历史仅作为 UI 提示或 BOSS 对话中的简短语境出现

## 时代（背景文字）

### 远古时代（基础语言）

> 开端之时，伟大的祖先 C 和 Unix 锻造了第一批修炼功法。他们的方法原始而强大——直接内存操作、指针算术、系统调用。这些古老的功法仍然是所有现代修炼的基础。

**现实映射**：1970-1980 年代，C 语言和 Unix 操作系统起源。

### 古典时代（面向对象崛起）

> 古典时代见证了大宗门的崛起——C++、Java 及其同类。他们引入"万物皆对象"的哲学，将修炼组织成结构化层级。Java 派在此期间崛起，宣讲跨所有平台的通用兼容。

**现实映射**：1990-2000 年代，Java 和 C++ 在企业中的主导地位。

### 现代时代（云与并发）

> 今日的修士面临新挑战——分布式系统、云架构、大规模并发。Go 派从 Google 仙山崛起以应对这些需求，宣讲简洁与效率。新宗门不断形成，各自提供自己的飞升之路。

**现实映射**：2010 年代至今，云原生、微服务、Go、Rust 崛起。

## 著名历史参考（BOSS 语境）

这些参考仅出现在 BOSS 记忆碎片中，不在一般游戏玩法中。

### 陈长老的时代

> *在陈长老年轻时，伟大的分布式系统正从无到有被锻造。他记得标准化协议前的混乱，无尽深夜调试竞态条件，第一个云平台的诞生。"我们建立了年轻修士现在视为理所当然的基础，"他反思道。"你使用的每个抽象，我们都必须发明。"*

### 刘清的时代

> *刘清的道路始于外包战壕——一种完全不同的修炼。她不在宏伟殿堂学习，而在实用工坊，在真实截止日期下修复真实系统。"声望门派教授优雅理论，"她说。"但必须有人让代码真正运行起来。"*

## 门派起源（简短风味）

### Java 派起源

> Java 派在古典时代从 Sun 宗崛起，后由 Oracle 联盟合并。其创始原则——"一次编写，到处运行"——挑战了当时的分裂局面。

### Go 派起源

> Go 派在向现代时代过渡期间诞生于 Google 仙山。其创始人寻求简化日益复杂的修炼功法。

## 历史内容不包括什么

| 排除 | 原因 |
|----------|-------|
| 具体日期或年份 | 保持隐喻，避免时代错乱 |
| 除 BOSS 外的历史人物 | 范围控制 |
| 战争或冲突 | 无门派竞争 |
| 详细事件编年史 | 与玩家旅程无关 |
| 改变世界的事件 | 焦点在玩家个人成长 |
