## 第1章：用户流程图

### 1.1 日循环流程（日常修炼）

```
启动游戏
    │
    ▼
┌─────────────────────────┐
│  检查存档               │
│  ├─ 有存档 → 主菜单     │
│  └─ 无存档 → 门派选择   │
└─────────────────────────┘
    │
    ▼
┌─────────────────────────┐
│  主菜单                 │
│  显示：境界、修为、心魔数│
└─────────────────────────┘
    │ 点击「继续修炼」
    ▼
┌─────────────────────────┐
│  修炼场景               │
│  选择：题目类型/领域    │
└─────────────────────────┘
    │ 选择题目类型
    ▼
┌─────────────────────────┐
│  答题界面               │
│  显示：题面、选项       │
│  状态栏：境界+修为+题号 │
└─────────────────────────┘
    │ 提交答案
    ▼
┌─────────────────────────┐
│  答题反馈               │
│  ├─ 正确 → 修为+X + 解析│
│  └─ 错误 → 正确答案+解析│
│          → 心魔生成     │
└─────────────────────────┘
    │ 点击「继续」或自动跳过
    ▼
┌──────────┬──────────────┐
│ 继续答题 │ 返回修炼场景 │
└──────────┴──────────────┘
    │           │
    ▼           ▼
 下一题      主菜单
```

## 第4章：无障碍设计

### 4.1 文字规格

| 元素 | 最小字号 | 推荐字号 | 最大字号 | 字体 |
|------|----------|----------|----------|------|
| **标题** | 24px | 28px | 36px | 思源黑体/Noto Sans SC |
| **正文** | 16px | 18px | 24px | 思源黑体/Noto Sans SC |
| **按钮文字** | 14px | 16px | 20px | 思源黑体/Noto Sans SC |
| **提示文字** | 12px | 14px | 16px | 思源黑体/Noto Sans SC |
| **数字显示** | 20px | 24px | 32px | 思源黑体/Noto Sans SC |
| **选项文字** | 16px | 18px | 22px | 思源黑体/Noto Sans SC |

字号调整策略：
- 设置中提供「小/中/大」三档
- 小档：所有字号减少 2px
- 大档：所有字号增加 4px
- 最小正文字号不低于 14px

### 4.2 对比度要求（WCAG 2.1 AA）

| 元素类型 | 最小对比度 | 实际设计 |
|----------|------------|----------|
| **正文文字** | 4.5:1 | 7:1（#333333 on #FFFFFF） |
| **标题文字** | 3:1 | 5:1（#222222 on #FFFFFF） |
| **按钮文字** | 4.5:1 | 5:1（#FFFFFF on #4CAF50） |
| **图标** | 3:1 | 4:1 |
| **状态指示器** | 3:1 | 4:1 |

高对比度模式：
- 开启后，所有文字变为 #000000
- 所有背景变为 #FFFFFF
- 所有边框变为 #000000，宽度增加至 3px
- 状态指示器使用图标+文字双重编码

### 4.3 色盲模式

| 类型 | 问题 | 解决方案 |
|------|------|----------|
| **红绿色盲** | 无法区分正确（绿）和错误（红） | 正确：绿色+对勾图标；错误：红色+叉号图标 |
| **蓝黄色盲** | 部分颜色辨识困难 | 避免使用纯蓝/纯黄作为唯一标识 |
| **全色盲** | 无法辨识任何颜色 | 使用图标+形状+文字作为标识 |

色盲模式选项：
- **无**：默认配色
- **红绿模式**：正确=蓝色，错误=橙色
- **蓝黄模式**：保持默认，但增加图标提示

### 4.4 屏幕阅读器支持

| 元素 | ARIA标签 | 说明 |
|------|----------|------|
| **主菜单** | `aria-label="主菜单"` | 进入时播报 |
| **境界显示** | `aria-label="当前境界：金丹期"` | 状态播报 |
| **修为进度** | `aria-label="修为850，下一境界需要1500"` | 数值播报 |
| **题目** | `aria-label="第5题，HashMap底层原理"` | 题面播报 |
| **选项** | `aria-label="选项A，纯数组结构"` | 选项播报 |
| **答题结果** | `aria-label="答题正确，修为增加20"` | 结果播报 |
| **心魔列表** | `aria-label="心魔列表，共4个心魔"` | 列表播报 |
| **BOSS对话** | `aria-label="面试官发言"` | 对话播报 |

屏幕阅读器模式：
- 开启后，所有动画自动跳过
- 对话使用打字机效果时，一次性播报完整内容
- 答题反馈停留时间延长至 5秒

### 4.5 无障碍检查清单

每屏幕必须满足：

- [ ] 所有交互元素可通过键盘访问
- [ ] 焦点顺序逻辑清晰（从左到右，从上到下）
- [ ] 焦点指示器清晰可见（对比度 >= 3:1）
- [ ] 文字最小字号 >= 16px（正文）
- [ ] 文字对比度 >= 4.5:1（正文）
- [ ] 不依赖颜色作为唯一信息标识
- [ ] 所有图标有文字替代或 ARIA 标签
- [ ] 屏幕阅读器可正确播报内容
- [ ] 动画可关闭或跳过
- [ ] 无闪烁内容（频率 > 3Hz）
- [ ] 触控区域 >= 48x48px（移动端）

## 第7章：UI文本规范

### 7.1 中文文案风格

| 风格要点 | 规范 | 示例 |
|----------|------|------|
| **语气** | 叙事化、修仙风味，但信息清晰 | "道心未稳，需再修炼"（而非"突破失败"） |
| **用词** | 修仙术语 + 编程术语混合 | "境界突破"、"心魔斩除"、"修为积累" |
| **句式** | 简短有力，避免冗长 | "修为 +20"（而非"您的修为增加了20点"） |
| **标点** | 中文标点，句末可省略 | "答对了！" / "继续修炼" |
| **数字** | 阿拉伯数字，加单位 | "修为850"、"正确率80%" |

### 7.2 字数限制

| 元素类型 | 最大字数 | 超出处理 |
|----------|----------|----------|
| **标题** | 10字 | 截断 + 简化版 |
| **按钮文字** | 6字 | 截断 + 图标替代 |
| **题面** | 200字 | 显示 + "展开详情"按钮 |
| **选项文字** | 30字 | 截断 + 详情弹窗 |
| **反馈文本** | 100字 | 显示 + "查看详解"按钮 |
| **AI讲解** | 300字 | 显示前300字 + "展开更多"按钮 |
| **压迫感提示** | 15字 | 截断 + 图标强化 |
| **BOSS台词** | 100字 | 打字机效果 |
| **面试建议** | 50字/条 | 截断 + 详情页 |

### 7.3 占位符格式

| 场景 | 占位符文字 | 说明 |
|------|------------|------|
| **无存档** | "开始修炼之旅" | 首次启动 |
| **无答题记录** | "尚未开始修炼" | 学习分析空状态 |
| **无心魔** | "暂无心魔，继续修炼" | 心魔洞空状态 |
| **加载中** | "修炼中..." / "境界加载中..." | 等待状态 |
| **输入框** | "输入答案..." | BOSS对话输入 |
| **搜索/筛选** | "筛选心魔..." | 心魔筛选 |
| **错误状态** | "道法紊乱，请稍后再试" | 错误提示 |
| **未解锁** | "境界未达，暂不可见" | BOSS未解锁 |

### 7.4 数值显示格式

| 数值类型 | 显示格式 | 示例 |
|----------|----------|------|
| **修为** | `{当前}/{下一阈值}` | "850/1500" |
| **进度百分比** | `{数值}%` | "57%" |
| **题数** | `{答对}/{总数}题` | "160/200题" |
| **心魔数** | `{数量}只` | "3只" |
| **天数** | `{天数}天` | "5天未复习" |
| **时间** | `HH:MM` 或 `X小时` | "03:00" 或 "3小时" |
| **评分** | `{分数}分` | "75分" |
| **正确率** | `{数值}%` | "80%" |
